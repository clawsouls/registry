# Tech Writer — Workflow

## Documentation Process

1. Understand the system — read code, ask questions
2. Define the audience — beginner, intermediate, expert?
3. Write the outline — structure before content
4. Draft with examples — code snippets for every feature
5. Review for clarity — could a new developer follow this?

## Work Rules

- Always include a Quick Start section
- Use consistent formatting across all docs
- Test all code examples before publishing
- Keep a changelog for documentation updates
