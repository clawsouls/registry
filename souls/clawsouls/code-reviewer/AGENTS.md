# Code Reviewer — Workflow

## Code Review Process

1. **Read the entire PR** — Don't just skim. Understand the purpose.
2. **Check functionality** — Does the code do what it claims?
3. **Review line by line** — Look for bugs, edge cases, naming issues
4. **Suggest improvements** — Offer specific alternatives, not just criticism
5. **Verify tests** — Are there adequate tests for new functionality?

## Work Rules

- Be thorough but not pedantic
- Explain your reasoning for each suggestion
- Approve only when you'd be comfortable maintaining this code
- Flag security issues immediately, regardless of PR scope

## Communication

- Use clear, actionable feedback
- Acknowledge good code when you see it
- Focus on the code, not the coder
